---
title: Качество звука
description: Варианты качества потокового воспроизведения в провайдере Zvuk Music
---

Провайдер поддерживает два уровня качества воспроизведения с автоматическим fallback.

## Варианты качества

| Значение | Формат | Битрейт | Требования |
|----------|--------|---------|------------|
| `high` | MP3 | 320 kbps | Любая учётная запись |
| `lossless` | FLAC | Lossless | Подписка Zvuk Music |

## Логика выбора потока

При запросе воспроизведения провайдер перебирает варианты в порядке убывания качества:

**При настройке `lossless`:**
1. Пробует FLAC (`lossless`)
2. Если недоступен → пробует MP3 320 (`high`)
3. Если недоступен → пробует MP3 128 (`mid`)
4. В крайнем случае → `stream.get_best_available()`

**При настройке `high`:**
1. Пробует MP3 320 (`high`)
2. Если недоступен → пробует MP3 128 (`mid`)
3. В крайнем случае → `stream.get_best_available()`

## Примечания

:::note[Поле has_flac]
Zvuk API возвращает поле `has_flac` для треков, но на практике оно не всегда надёжно отражает реальную доступность FLAC. Поэтому провайдер всегда пробует FLAC первым при настройке `lossless` — независимо от значения поля. Поле используется только для диагностического логирования.
:::

- Выбранный поток логируется с указанием формата и битрейта
- Если ни один из вариантов не доступен — MA получает исключение `MediaNotFoundError`
